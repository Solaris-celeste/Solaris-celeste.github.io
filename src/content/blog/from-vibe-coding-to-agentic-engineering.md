我们大概都经历过这个阶段：

最早是把 AI 当“补全工具”，后来变成“结对程序员”，再后来，很多人开始尝试让它连续完成一个任务链——读代码、改代码、跑测试、回滚、再改。这个变化看起来只是效率提升，但我越来越觉得，它在悄悄改写科研的基本单位。

以前我们讨论“会不会写代码”，现在真正的分水岭变成：你能不能把一个模糊研究目标，压缩成机器可执行、可验证、可迭代的任务流。

![从 vibe coding 到 agentic engineering 的演进示意图](../../assets/blog-placeholder-3.jpg)

## 1) Vibe Coding 的价值，不在“快”，在“低门槛探索”

很多人对 vibe coding 的理解停在“糊代码很爽”。这当然没错，但不够准确。

它更重要的意义是：让“想法 -> 原型”这一步骤变得便宜。

在科研里，很多好问题死在原型前——不是因为问题不重要，而是因为实现成本太高、反馈周期太长。你要搭环境、啃接口、补脚手架，最后还没来得及验证核心假设，时间就被基础工程吞掉了。

vibe coding 的真正收益是把这些前置成本打薄。你可以在很短时间内得到一个“粗糙但可跑”的版本，把研究从“想象争论”推进到“可观察结果”。这对早期探索非常关键。

但它也天然有边界：

- 它擅长局部生成，不擅长全局一致性；
- 它擅长补齐结构，不擅长保证研究问题被正确表达；
- 它能让你很快跑起来，也能让你很快跑偏。

换句话说，vibe coding 是“加速器”，不是“方向盘”。

## 2) Agentic Engineering 真正改变的是“科研节拍”

当我们从单次生成，进入多步自治执行（agentic engineering），变化就不只是快一点了，而是科研节拍被重排。

一个典型的 agent 流程大概是：

1. 读取目标与约束；
2. 分解任务并生成执行计划；
3. 调用工具链（代码、实验、检索、评估）；
4. 根据反馈自动修正并继续。

这和传统“人手工推进每一步”相比，最大的差异是：

- **并行化程度大幅提升**：多个候选方向可以同时试错；
- **迭代闭环更短**：从 hypothesis 到结果，不必每一步都人工介入；
- **工程-研究边界变模糊**：很多原本“实现细节”开始直接影响研究结论。

![Agent loop 如何压缩科研反馈周期](../../assets/blog-placeholder-4.jpg)

这会带来一个现实后果：

未来最有竞争力的研究者，不一定是“写得最快的人”，而是“能设计出高质量反馈回路的人”。

也就是：你是否能定义清楚成功标准、失败信号、停止条件、对照实验和异常处理。Agent 可以替你跑很多步，但“跑到哪里停、为什么停、结果是否可信”，仍然是人要负责的。

## 3) 技术上最容易被忽略的，不是模型能力，而是评估结构

今天很多讨论都围绕“模型又变强了多少”。我反而觉得，科研场景里更关键的是评估结构是否成熟。

因为 agent 能持续行动，**错误也会持续放大**。如果你的评估面是单薄的，系统会在错误目标上疯狂优化。

几个高频风险非常典型：

- **代理指标绑架**：优化了容易算的指标，却偏离真正科学问题；
- **数据泄漏与实验污染**：在多轮自动化中悄悄发生，最后得到“漂亮但虚假”的结论；
- **可复现性坍塌**：每一步都“看起来合理”，但回放不到同样结果；
- **解释漂移**：结果是对的，解释却是错的，导致后续研究基于错误叙事继续搭建。

所以我越来越相信：

> Agentic engineering 的上限，不由生成能力决定，而由评估能力决定。

如果把科研看作一个系统工程，那么模型是执行器，评估框架才是控制器。

## 4) 当“会做”不再稀缺，人类研究者的价值会回到哪里？

我想了很久这个问题。一个朴素但坚定的结论是：

**人的意义不是和 AI 比产出速度，而是为知识生产提供方向、边界与责任。**

具体来说，我理解未来研究者至少承担四种不可替代的角色：

### 4.1 问题定义者（Problem Framing）

真正高价值的问题，通常不是从数据里“自动挖”出来的，而是来自对领域矛盾、社会需求和技术边界的综合判断。这个判断高度依赖经验、价值观和语境理解。

### 4.2 证据审计者（Evidence Auditor）

AI 可以给你结果，但“证据链是否成立”需要人去审计：

- 变量控制是否充分？
- 对照是否公平？
- 结论外推是否越界？
- 负结果是否被忽略？

科研不是生成答案，而是约束自己不要轻易相信答案。

### 4.3 范式连接者（Paradigm Integrator）

很多突破来自跨领域迁移：把 A 领域的方法借给 B 领域的问题。Agent 擅长在既有空间里搜索，人更擅长改写搜索空间本身。

### 4.4 伦理与后果承担者（Responsibility Holder）

当研究结果进入现实世界，影响的是人而不是 benchmark。这里涉及安全、公平、资源分配、误用风险，最终都需要由人承担解释与责任。

## 5) 我心里更激进的一点：我们可能要重写“科研训练”

如果以上判断成立，那么“培养一个研究者”的路径也该变化。

过去我们花大量时间训练执行能力——实现、调参、复现。它们仍然重要，但不再足够。

未来更应该强化的是：

- **问题表达能力**：把模糊问题转成可验证命题；
- **评估设计能力**：搭建可靠、可复现、可审计的验证框架；
- **系统思维能力**：理解数据、模型、工具、部署之间的耦合关系；
- **价值判断能力**：知道“能做”和“该做”之间差了什么。

![研究者能力重心迁移：从执行到判断](../../assets/blog-placeholder-5.jpg)

说得直白一点：

以后“写代码”可能像今天“会用搜索引擎”一样，成为基础能力；而真正稀缺的，是你是否能提出一个值得世界花算力去验证的问题。

## 结语

从 vibe coding 到 agentic engineering，我看到的不是“人被替代”的单线故事，而是“人被迫升级”的现实。

机器正在接管越来越多的执行环节，这并不自动让科研更真诚、更可靠、更有意义。恰恰相反，它要求研究者在更高层面上变得更严格：对问题更严格、对证据更严格、对后果更严格。

如果说过去科研的核心是“把事做成”，那未来科研的核心也许会变成：

**在极高的自动化能力面前，仍然坚持把对的事做成。**
